添加本地项目到github
1.打开终端检查本地是否安装github，ubuntu默认是安装github的。 
git --version
2.生成密钥对，此步骤如果遇见输入密码时，连续按回车键即可。
ssh-keygen -C "itinterpret@gmail.com" -f ~/.ssh/github
3.查看公钥。
cat ~/.ssh/github.pub
4.登录github账号：
    • 点击头像 
    • 选择Settings 
    • 点击左边对话框的SSH and GPG keys 
    • 点击右上角的New SSH key 
    • 此时会出现如下界面： 
      将第3步中列出的公钥拷贝到上图的Key区域，Title任意。
5.检查本地是否能远程访问github服务器，出现You’ve successfully authenticated，说明认证通过。
ssh -T git@github.com
6.创建本地github库，注意创建本地库时一定要在github上也创建一个相同的库。
mkdir github_repository//建立一个文件夹，作为本地库的名字，将要提交的东西放到这个本地库中。
git init //初始化本地的github仓库

git config --global user.name "itInterpret"                 #"github的用户名" 
git config --global user.email "itinterpret@gmail.com"      #"注册邮箱名"

7.提交本地项目到github，首先是添加提交信息，这一步可能会出现下图的错误。
git add "文件/文件夹名称"
git commit -m "提交说明"//提交项目的说明12
如果出现上图中的错误，执行下面这个命令：
git config --global user.email "邮箱号"

9.如果第8步执行的时候，报如下图的错：  
可以执行下面的命令，然后重新提交。
git remote add origin git@github.com:itInterpret/IPEYE_PLATES_VL.git

###
git remote rm origin 

git pull --rebase origin master

8.提交本地项目到远程github，出现下图则提交成功，登录github账号后在相应的库中可以看到提交的本地代码。
git push -u origin master//提交到github

出现错误的主要原因是github中的README.md文件不在本地代码目录中
如何解决failed to push some refs to git
可以通过如下命令进行代码合并
pull=fetch+merge
git pull --rebase origin master
